## Overview

Crawl and add websites to an agent's knowledge base. This endpoint allows you to specify a base URL and various crawling parameters to extract content from websites and make it available to your agent for knowledge-based responses.

## Authentication

This endpoint requires API key authentication via the `X-Authorization` header.

## Crawling Parameters

### URL Configuration

- **url** (required): The base URL to start crawling from
- **includePaths**: Array of URL paths to specifically include (whitelist approach)
- **excludePaths**: Array of URL paths to exclude from crawling (blacklist approach)

### Crawling Limits

- **limit**: Maximum number of pages to crawl (default: 5, max: 10)
- **maxDepth**: Maximum crawling depth from the base URL (default: 1, max: 10)

## Content Processing

The system automatically:

1. Crawls the specified website(s) based on your parameters
2. Extracts and processes the content using Markdown conversion
3. Stores both JSON and Markdown formats for optimal retrieval
4. Associates the content with your agent's knowledge base

## Use Cases

- **Documentation Sites**: Add product documentation or help centers
- **Company Websites**: Include company information and policies
- **Knowledge Bases**: Import existing knowledge articles
- **Blog Content**: Add relevant blog posts or articles
- **FAQ Pages**: Include frequently asked questions

## Best Practices

1. **Start Small**: Begin with a low limit and maxDepth to test crawling
2. **Use Path Filters**: Employ includePaths/excludePaths to focus on relevant content
3. **Monitor Limits**: Stay within the 10-page and 10-depth limits for optimal performance
4. **Content Quality**: Ensure the target website has structured, readable content

## Error Handling

- `400`: Invalid request parameters (e.g., malformed URL, invalid limits)
- `401`: Unauthorized (invalid API key or insufficient permissions)
- `500`: Crawling errors or content processing failures
